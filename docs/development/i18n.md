## Service Implementation

I18nService handles:
- Loading translation files
- Current language state
- Translation lookup with dot notation
- Parameter interpolation
- LocalStorage persistence

---

## TranslatePipe

Use in templates:

```html
<!-- Simple translation -->
<h1>{{ 'boards.title' | translate }}</h1>

<!-- With parameters -->
<p>{{ 'validation.minLength' | translate: { min: '3' } }}</p>

<!-- In attributes -->
<button [attr.aria-label]="'common.close' | translate">X</button>
```

---

## Language Switcher Component

Create a dropdown or toggle in header:

```html
<select (change)="onLanguageChange($event)">
  <option value="en">English</option>
  <option value="ru">Русский</option>
</select>
```

Component handles calling `i18nService.setLanguage(lang)`.

---

## Translation Keys Structure

Use hierarchical keys:

```
{feature}.{component}.{key}
common.{key}
validation.{key}
```

**Examples**:
- `common.save`
- `auth.login`
- `boards.create`
- `validation.required`

---

## Parameter Interpolation

Format: `{paramName}`

**Translation**:
```json
{
  "greeting": "Hello, {name}!"
}
```

**Usage**:
```html
{{ 'greeting' | translate: { name: userName() } }}
```

**Result**: "Hello, John!"

---

## Initial Language Detection

Priority order:
1. LocalStorage value
2. Browser language (`navigator.language`)
3. Default: `en`
